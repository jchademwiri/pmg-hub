# PMG Hub User Guides

> Start here if you are new to the system. These guides explain what each area is for, what to do day to day, and how the information flows through the business.

---

## The Big Picture

PMG Hub is the admin system for running the business. It connects client relationships, quotes, invoices, payments, expenses, accounting, monthly snapshots, reports, settings, exports, and backups.

Most users do not need to understand every accounting detail. The practical flow is:

```text
Relationships -> Quotes -> Invoices -> Payments -> Income
                                      -> Expenses
                                      -> Accounting
                                      -> Reports and Snapshots
```

Use the operational pages first. The accounting pages are mainly for review, correction, and accountant support.

---

## Your First Day

1. Open the dashboard and check the current month.
2. Confirm which division you are working in:
   - `AWS` - Apex Web Solutions
   - `TES` - Tender Edge Solutions
   - `PMG` - Playhouse Media Group
3. Add or check the client in Relationships.
4. Create a quote if the work is not approved yet.
5. Create or issue an invoice when the work is ready to bill.
6. Record payments when money arrives.
7. Record expenses when money leaves the business.
8. Use Reports and Snapshots to review performance.

## Daily Routine

| Task | Where To Go | Notes |
|------|-------------|-------|
| Add a new client | Relationships -> Clients | Do this before billing them |
| Send a quote | Billing -> Quotes | Use the correct division |
| Issue an invoice | Billing -> Invoices | Drafts are not sent yet |
| Record a payment | Billing -> Payments | Allocate it to the right invoice |
| Record an expense | Finance -> Expenses | Choose the best category |
| Check what is owed | Billing -> Statements or client billing | Useful for follow-ups |
| Review performance | Dashboard or Insights -> Reports | Use actual numbers, not memory |

---

## Month-End Routine

1. Make sure all payments and expenses for the month are entered.
2. Check draft invoices and decide whether to issue, keep, or remove them.
3. Review the Trial Balance and Profit & Loss.
4. Close the month from the dashboard when the previous month is ready.
5. Review the locked month in Insights -> Snapshots.
6. Confirm backups are running in Settings -> Data.

Important: if you are in June, June is still open. The close-month flow should usually close May, because it locks the completed month.

---

## Common Rules

- Use `AWS`, `TES`, and `PMG` consistently when checking division-specific records.
- Do not delete records just because something changed. Void, cancel, credit, or correct them so the audit trail remains clear.
- Record payments only after money actually arrives.
- Record expenses with clear descriptions, not vague labels like "misc".
- Close months only after the numbers have been reviewed.
- Ask an admin before changing Settings, Security, Data, or Backup configuration.
